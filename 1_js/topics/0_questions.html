1. 如何进行前端性能优化 a. 尽量能不做的事情就不做 1. 图片、组件的懒加载 2. 页面分页 3. 减少网络请求：代码内存中处理是很快的，网络请求、数据库查询相对是比较费时间的 3.1 减少htpp请求 3.2 减小请求时文件大小(图片压缩，代码压缩webpack打包之后，文件小了，不需要的包也没有) 3.3 4. 减少dom操作，特别是频繁操作真实dom b. 即便要做同一件事情，尽量以更小的代价完成 代码层面，算法要精简，更低的时间复杂度、空间复杂度 2. 一次完整的http请求 a. dns -> ip地址 b. 三次握手进行tcp连接 c. 客户端发送请求地址和参数 d. 服务器nginx根据contentPath转发给对应的服务和端口，根据url path，匹配到controller server - dao - 数据库 - 处理 - 返回给客户端 e1. dom, css -> 浏览器分开渲染dom树，css样式 e2. restful接口，数据接收后，调用回调函数执行
